### Apprentissages / pièges pour sessions futures

1. **Telegram et pièces jointes inter-sessions** : quand Paul envoie une image via un message user standard, le runtime ne la stocke que comme référence textuelle (`<media:image>`). Pour qu'elle soit lisible par `read`, il faut que l'image arrive via une **pièce jointe de tool result** (image_generate, web_fetch, etc.). Paul a dû renvoyer 2x pour que ça passe.

2. **Banana prompts pour icônes** : `minimax/image-01` ignore `outputFormat=png` et `background=transparent`. Toujours supposer JPEG opaque, ou utiliser SVG puis `@resvg/resvg-js` pour la conversion.

3. **PowerShell `curl` alias** : c'est `Invoke-WebRequest` qui demande `-SessionVariable`. Pour un vrai curl : `C:\Windows\System32\curl.exe`.

4. **Windows paths et tar.exe** : `tar -czf out.tar.gz -C "$PWD" .` plante sur les chemins avec espaces. Workaround : `cd` dans le dossier puis `tar -czf out.tar.gz .`

5. **Auth.js v5 + Next 14** : `next-auth/jwt` deprecated en v5, remplacé par `auth()` helper. Le code livré utilise encore `getToken()` car il marche, mais Phase 2 = migrer.

6. **Serwist types** : `ServiceWorkerGlobalScope` n'est pas dans lib "DOM" par défaut. Workaround : `// @ts-ignore` ou ajouter `"WebWorker"` dans tsconfig lib.
